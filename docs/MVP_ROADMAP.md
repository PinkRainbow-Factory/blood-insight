# MVP 로드맵

## 1. 목표
- 8~12주 안에 "동작하는 Android MVP"를 만든다.
- 첫 버전은 가장 중요한 입력, 해석, 리포트 경험에 집중한다.

## 2. 단계별 범위
### Phase 1. 설계 및 기반
- PRD 확정
- 정보구조 설계
- 디자인 시스템 초안
- 기술 스택 확정

### Phase 2. 코어 백엔드
- 인증
- 프로필 저장
- 질환 코드 검색 API
- 검사결과 저장 API
- 기준치 비교 엔진

### Phase 3. 코어 프론트
- 온보딩
- 일반/환자 모드 분기
- 수동 입력 화면
- 리포트 화면
- 항목 팝업

### Phase 4. AI 해석
- OpenAI/Gemini 공급자 레이어
- 프롬프트 템플릿
- 구조화 결과 -> 설명문 변환
- 안전 필터 적용

### Phase 5. OCR
- 이미지 업로드
- OCR 결과 매핑
- 사용자 검수 화면

### Phase 6. 고도화
- 추세 그래프
- PDF 내보내기
- 애니메이션 강화
- 운영 어드민 도구

## 3. MVP 포함 항목
- Android 11+
- 이메일 로그인
- 프로필
- 환자/일반인 분기
- 질환 검색
- 수동 입력
- 이미지 업로드
- 핵심 혈액검사 항목 분석
- AI 리포트
- 저장된 기록 보기

## 4. MVP 제외 항목
- 의료진 전용 대시보드
- 실시간 채팅 상담
- EMR 직접 연동
- 다국어
- 보험 연계

## 5. 우선순위
### 반드시 먼저
- 정확한 데이터 입력
- 기준치 비교 엔진
- 질환 맥락 반영
- 안전한 AI 응답

### 다음 단계
- OCR 정교화
- 추세 시각화
- 화려한 애니메이션
- PDF 공유

## 6. 팀 구성 권장
- PM/기획
- 모바일 프론트엔드
- 백엔드
- AI 엔지니어
- 디자이너
- 의학 자문

## 7. 다음 구현 시작점
1. `React + Capacitor` 앱 뼈대 생성
2. 로그인/온보딩/프로필 화면 제작
3. 질환 검색과 수동 입력 폼 제작
4. 정적 HTML 리포트 템플릿 제작
5. 이후 AI/백엔드 연결
