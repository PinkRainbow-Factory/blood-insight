# AI 의료 해석 엔진 설계

## 1. 목표
- 혈액검사 수치를 질환 및 개인 프로필 맥락에서 해설한다.
- AI는 자연어 설명을 담당하고, 핵심 판단은 구조화 엔진에서 수행한다.
- 상용 앱 수준의 일관성, 안전성, 추적 가능성을 확보한다.

## 2. 엔진 구성
### 2.1 입력 계층
- 사용자 프로필
- 환자/일반인 구분
- 질환 코드 및 질환명
- 복용약
- 검사일
- 혈액검사 항목 목록
- 기준치 정보
- OCR 신뢰도

### 2.2 구조화 분석 계층
- 항목 정규화
- 기준치 비교
- 중요 항목 점수화
- 질환 관련성 매핑
- 다중 항목 패턴 분석

### 2.3 AI 설명 계층
- 전체 요약
- 항목별 설명
- 질환과의 관계 설명
- 생활관리 팁
- 진료실 질문 추천

### 2.4 안전 계층
- 금지 표현 필터
- 응급 문구 필터
- 진단/치료 지시 차단
- 불확실성 고지

## 3. 분석 방식
## 3.1 룰 기반 + AI 하이브리드
- 룰 엔진이 아래 항목을 계산:
  - 정상/경계/이상
  - 상승/감소 방향
  - 질환 연관도
  - 우선 확인 필요도
- AI는 아래 항목을 생성:
  - 쉬운 설명
  - 맥락화된 요약
  - 사용자가 이해하기 쉬운 관리 포인트

## 3.2 예시 해석 흐름
1. `WBC`, `Hb`, `Platelet`, `CRP`, `AST`, `ALT` 등 항목 수집
2. 기준치와 비교
3. 혈액암 환자 여부 확인
4. 혈액암과 강한 관련이 있는 항목 우선 정렬
5. AI가 "이 수치가 왜 중요하고 무엇을 시사하는지" 설명
6. 직접적인 치료 변경 지시는 피하고 의료진 상담 포인트 제시

## 4. 질환 매핑 전략
### 4.1 코드 체계
- 1차 권장: ICD-10
- 한국 시장 최적화 시 KCD 매핑 추가

### 4.2 질환 지식 구조
- 질환 기본 설명
- 중요 혈액검사 항목 목록
- 상승 시 의미
- 감소 시 의미
- 추적에 자주 쓰이는 항목
- 경고 신호 문구

### 4.3 예시 질환군
- 혈액암
- 빈혈
- 간질환
- 신장질환
- 갑상선질환
- 염증성 질환
- 당뇨/대사질환

## 5. 응답 구조 표준
```json
{
  "overall_summary": "전체 요약",
  "priority_items": [
    {
      "test_code": "HGB",

      "status": "low",
      "severity": "medium",
      "why_it_matters": "현재 질환 맥락에서 중요한 이유",
      "patient_friendly_explanation": "쉬운 설명",
      "care_tip": "일반적 관리 포인트",
      "ask_doctor": "진료실에서 물어볼 질문"
    }
  ],
  "safety_notice": "본 결과는 의료 상담을 대체하지 않습니다."
}
```

## 6. 프롬프트 아키텍처
### 6.1 시스템 프롬프트 역할
- 의료 교육용 해설자로 동작
- 구조화 데이터를 벗어난 임의 추론 최소화
- 사용자 친화적 설명
- 과도한 확정 표현 금지

### 6.2 개발자 프롬프트 역할
- 출력 형식 고정
- 금지 문장 정의
- 응급/진단/처방 관련 제한
- 사용자 유형별 톤 조정

### 6.3 사용자 입력 역할
- 개인 프로필
- 질환 정보
- 혈액검사 값
- 사용자가 원하는 설명 수준

## 7. 금지/제한 정책
- "당신은 이 질병입니다" 같은 확정 진단 금지
- 약 복용 변경 지시 금지
- 응급실 방문 강제 표현은 위험 신호 조건 충족 시에만 제한적으로 사용
- 불충분한 정보에서 과도한 결론 금지

## 8. AI 공급자 추상화
- Provider Interface:
  - `generateSummary(input)`
  - `generateItemExplanation(input)`
  - `generateQuestionList(input)`
- 공급자:
  - OpenAI Provider
  - Gemini Provider
- 공통 응답 스키마 유지

## 9. 품질 관리
- 의학 자문 기반 테스트셋 구축
- 질환군별 예시 케이스 검증
- 환각 사례 수집
- 위험 문장 자동 탐지
- 응답 로깅 및 리뷰 시스템 구축

## 10. 개인화 요소
- 연령/성별
- 진단 질환
- 치료 상태
- 복용약
- 이전 검사 추세
- 검사 목적(정기검진, 치료 중 추적 등)

## 11. 리포트 UI 연결 데이터
- 상태 색상
- 중요도 배지
- 항목별 설명 텍스트
- 관련 질환 태그
- 주의 포인트 카드
- 질문 추천 카드

## 12. MVP 기준 추천 범위
- 단일 검사 회차 분석
- 핵심 항목 중심
- 단일 또는 소수 질환 매핑
- 한국어 우선
- HTML 리포트 1종 템플릿
